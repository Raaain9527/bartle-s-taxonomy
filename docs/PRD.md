# 玩家类型测试 — 产品需求文档

## 1. 产品概述

### 1.1 产品定位
一款 SBTI 调性的娱乐化游戏玩家类型测试工具。以"不正经"的自嘲口吻，通过 24 道情景选择题，将玩家归类为 12 种游戏人格之一，并生成可分享的诊断海报。

### 1.2 核心差异化
- **SBTI 式语调**：戏谑、自嘲、拒绝说教，标签让人想截图转发
- **小黑怪诞视觉**：基于 Ian Xiaohei 风格的角色插画，纯白背景、黑色手绘线稿、少量批注色
- **学术暗线**：底层基于 Bartle 四象限 + Quantic Foundry 动机维度，分类有依据但不外露
- **零门槛**：纯静态站，24 题 3 分钟，做完即出结果

### 1.3 目标用户
- 18-30 岁游戏玩家
- 活跃于小红书/朋友圈/B站/微博的社交分享者
- 对 MBTI 审美疲劳、对 SBTI 调性有认同感的年轻人

## 3. 题库设计

### 3.1 题量与结构
- **24 题**，每题 3 个选项（A/B/C）
- 完成时间：2-4 分钟
- 题型：游戏情景选择题，每题描述一个具体游戏场景

### 3.2 计分规则
- 每题选择对应的类型 +2 分（与 v1 一致，暂不做软化计分）
- 24 题结束后，取最高分类型为最终结果
- 同分时：比较第二高分的 Bartle 象限归属，取与该象限更一致的类型
- 极端同分（两类型分数完全一致）：展示"双重人格"结果页，同时展示两个类型

**严重程度计算**（结果页进度条）：
```
浓度 = 最高分 / 满分(12) × 100%
```
- ≥ 83%：重度（"建议入院观察"）
- 67-82%：中度（"临床症状明显"）
- 50-66%：轻度（"偶发症状"）
- < 50%：边缘（"你可能是个正常人，再做一遍"）

**类型稀有度**（模拟数据，非真实统计）：
- SSR（超稀有，占玩家 3-5%）：考古学家、白金人
- SR（稀有，占玩家 6-10%）：BUG启动器、游戏奶妈、该溜子、氪佬
- R（常见，占玩家 11-18%）：分奴、压力怪、乐子人、赛博驴、仓鼠、挂机仙人
- 稀有度标签仅用于增加结果页戏剧性，不参与计分逻辑

### 3.3 题目设计原则
- **情景化**：每道题是具体的游戏场景，不直接问"你是什么样的人"
- **选项平等**：三个选项没有明显的"好/坏"之分，避免社会赞许偏差
- **幽默感**：选项措辞带轻吐槽，让用户边选边笑
- **覆盖度**：12 种类型各出现 6 次（24 题 × 3 选项 = 72，72 ÷ 12 = 6）

### 3.4 平行题库机制（蛋仔派对）

主测试 Q1 含有一个"蛋仔触发器"选项，选择后自动跳转到蛋仔派对专属题库。

```
主题库 Q1-C（派对休闲选项）→ redirect: "eggy" → 前端加载 questions-eggy.json
蛋仔题库 Q1-C（开心就好选项）→ redirect: "main" → 前端加载 questions.json
```

**实现规则**：
- 选项 JSON 中增加 `redirect` 字段，前端检测到后切换题库文件
- 两套题库共享相同的 12 类型体系、计分引擎和结果页
- 蛋仔题库同为 24 题 × 3 选项，12 类型各 6 次
- 蛋仔题库所有题目围绕蛋仔派对世界观（竞速关、蛋仔岛、盲盒、乐园工坊等）
- 前端需在答题开始时记录当前使用的题库 ID，结果页据此展示不同的"代表游戏"推荐

### 3.5 答题交互规则
- **不可回退**：选择后自动跳下一题，不允许返回修改（增强"命中注定"感 + 防止用户反复修改追求特定结果）
- **必答所有题**：24 题全部完成才展示结果，不可提前提交
- **点击即提交**：点击选项后 300ms 内触发过渡动画进入下一题，无需二次确认
- **中途离开**：刷新/关闭页面后答题进度丢失（纯前端无存储，也防止"刷结果"）

## 5. 技术方案

### 5.1 技术栈
- **纯静态站**：HTML + CSS + Vanilla JS，无框架，无构建步骤（MVP）
- **构建工具**：无需构建（MVP 阶段）。后续若引入构建，Vite 为首选
- **部署**：GitHub Pages / Vercel / 腾讯云 COS 静态托管
- **依赖**：html2canvas（海报生成）、可选 countapi 做模拟计数
- **CSS 方案**：CSS 自定义属性做设计令牌管理；CSS Grid + Flexbox 布局；`clamp()` 做流体排版
- **暗色模式**：CSS 自定义属性 + `[data-theme="dark"]` 选择器 + `localStorage` 持久化
- **动效**：CSS transitions/animations 为主，避免引入动画库

### 5.2 数据流
```
questions.json → 浏览器加载 → 用户答题 → 本地计分 → 匹配类型 → 渲染结果页
```
全程无后端，无数据上传，隐私安全。

**答题状态机**：
```
IDLE → 加载题库 → 答题中(Q1→Q24) → 计分 → 匹配类型 → 展示结果
                    ↑                                  │
                    └── 检测到 redirect ──→ 切换题库 ──┘
```

## 6. 分享机制

### 6.1 海报生成

**竖版海报（主分享格式）**：
- 尺寸：1080 × 1920 px（适配小红书 / 朋友圈 / Instagram Story）
- 内容布局（从上到下）：
  1. 顶部：小黑角色插画（16:9 裁剪或完整展示）
  2. 中部：类型标签大字 + 稀有度标签（SSR/SR/R）
  3. 下部：一句话人设 + 口头禅 + 网站二维码
  4. 底部：版权声明（"Character IP based on Ian Xiaohei by @helloianneo"）
- 使用 html2canvas 将隐藏的结果卡片 DOM 渲染为图片
- 手机端长按保存，桌面端一键下载

**横版 OG 图（链接预览）**：
- 尺寸：1200 × 630 px
- 用于 `<meta property="og:image">` 标签
- 内容：小黑 + 网站标题 + 一句话介绍，横版布局

### 6.2 文字分享
- 预设分享文案："测了，我是 [类型标签]（[稀有度]级稀有度）。我的游戏人格口头禅是：'[口头禅]'。你也来测测？[链接]"
- 一键复制文案 + 链接到剪贴板
- 使用 Web Share API（移动端优先），桌面端回退到剪贴板复制

## 7. 设计基调

### 7.1 色彩

**亮色模式**
| 用途 | 色值 | 说明 |
|------|------|------|
| 背景 | `#FEFEFE` | 微暖白，略像纸张 |
| 卡片底 | `#F5F5F0` | 极浅纸色 |
| 主文字 | `#1A1A1A` | 近黑 |
| 次文字 | `#8C8C8C` | 灰 |
| 强调/警告 | `#D63031` | 红 — 批注、诊断标记、重点 |
| 路径/流程 | `#E17055` | 橙 — 箭头、进度、连接 |
| 辅助注释 | `#5B7FA5` | 蓝灰 — 仅次要标注 |

**暗色模式**
| 用途 | 色值 | 说明 |
|------|------|------|
| 背景 | `#1A1A1A` | 深黑 |
| 卡片底 | `#2A2A2A` | 深灰卡片 |
| 主文字 | `#EDEDED` | 浅白 |
| 次文字 | `#999999` | 中灰 |
| 强调/警告 | `#FF6B6B` | 红（暗色下提亮） |
| 路径/流程 | `#F0A58D` | 橙（暗色下提亮） |
| 辅助注释 | `#7BA5C8` | 蓝灰（暗色下提亮） |

### 7.2 字体
- 中文：系统默认中文字体栈（苹方 / 微软雅黑 / 思源黑体）
- 批注/手写感文字：使用内嵌手写字体或 CSS 模拟手写效果
- 数字/代码：系统等宽字体

### 7.3 视觉风格
- 大量留白，每屏只聚焦一件事
- 手绘线条感装饰元素（用 CSS border/border-image 模拟手绘线段，避免引入额外图片资源）
- 小黑角色贯穿各页面作为视觉锚点
- 拒绝渐变、阴影、圆角过大的"产品设计感"
- **圆角策略**：卡片 0（直角纸条感），按钮 2px（微圆），标签 0（诊断贴纸感）
- **阴影体系**：不用真实 box-shadow，用细线/边框表达层次（`0 1px 0 #E0E0E0`），呼应手绘感

### 7.4 动效规范
- **选项点击 → 下一题**：120ms `scale(0.98)` 按压反馈 → 250ms `ease-out` 左滑出 + 新题右滑入
- **进度条**：400ms `ease-out` 宽度过渡
- **结果页入场**：逐元素 staggered 淡入（各元素间隔 80ms，总计 ~600ms）
- **按钮/卡片 hover**：200ms 背景色/边框色过渡
- **全局 easing**：`cubic-bezier(0.4, 0, 0.2, 1)`（Material Design 标准缓动）
- **暗色模式切换**：300ms 全局颜色过渡（`transition: background-color, color`）
- **不做**：弹跳、旋转、脉冲、粒子效果（保持克制，与手绘冷静风格一致）

### 7.5 OG 标签与 SEO
首页需定义以下 `<meta>` 标签，用于社交链接预览：
```html
<meta property="og:title" content="游戏人格测试 — 你是什么玩家？" />
<meta property="og:description" content="24题，3分钟，测出你的12种游戏人格之一。SBTI式荒诞诊断，不正经，但很准。" />
<meta property="og:image" content="https://[域名]/assets/images/og-image.png" />
<meta property="og:image:width" content="1200" />
<meta property="og:image:height" content="630" />
<meta property="og:type" content="website" />
<meta name="twitter:card" content="summary_large_image" />
```
结果页需动态更新 `og:title`（替换为"测了，我是[类型标签]"）以支持个性化分享预览。

## 8. MVP 范围

### 包含
- 首页 + 答题页 + 结果页（12 类型 + 双重人格）+ 类型一览
- 主题库 24 题 + 蛋仔平行题库 24 题 + 计分引擎 + 题库切换逻辑

- 移动端优先响应式适配
- OG 标签 + 社交分享预览

### 不包含
- 用户系统 / 登录
- 排行榜 / 数据统计
- 后端 API
- 国际化（中英双语）
- 管理后台
- 结果对比 / 好友匹配
